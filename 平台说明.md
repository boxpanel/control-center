# 综合管理平台说明

## 1. 概览

本平台包含两类能力：

- **独立使用**：只提供本机 Web UI、摄像头事件接收（HTTP/FTP）、串口转发、数据落库等能力，不参与平台发现与上报。
- **客户端模式（可被管理）**：开启后，设备支持“搜索发现协议（UDP Probe/Match）”并可向注册中心上报（register/heartbeat），便于一对多管理。

客户端模式开关：`system.clientMode`

## 2. 设备搜索协议（UDP Probe / Match）

### 2.1 目的

用于“服务端/管理端”在局域网内发现设备，并拿到设备的基础信息（设备名称、可访问的 httpBase、IP 列表等）。

### 2.2 开关与生效条件

- 仅当设备 **客户端模式开启**（`system.clientMode = true`）时，才会启用该协议并可被搜索到。
- 客户端模式关闭时：设备不响应 Probe（独立使用，避免被扫描到）。

### 2.3 网络参数（默认值）

- **组播地址**：`239.255.255.250`
- **UDP 端口**：`10086`
- 可配置字段：`probe.group` / `probe.port` / `probe.enabled`

说明：
- 该端口已从常见的 `3702` 调整为 `10086`，以避免与 ONVIF/WS-Discovery 的常用端口混用导致排障混淆或端口竞争。

### 2.4 数据编码

- UDP 数据为 **UTF-8 JSON 文本**（不是 XML/SOAP）。

### 2.5 请求包（Probe）

**最小请求字段：**

```json
{ "t": "probe" }
```

**可选字段：**

- `replyPort`：设备回包端口（未填则回包到发包方端口）
- `registryBaseUrl`：如果携带有效 http/https 地址，设备会保存并用于后续注册上报

示例：

```json
{
  "t": "probe",
  "replyPort": 10087,
  "registryBaseUrl": "http://192.168.1.10:8080"
}
```

### 2.6 响应包（Match）

设备收到 Probe 后回包 Match：

```json
{
  "t": "match",
  "clientId": "设备指纹(sha256)",
  "name": "设备名称",
  "installDate": "YYYY-MM-DD",
  "ipList": ["192.168.1.22"],
  "service": { "httpBase": "http://192.168.1.22:3000" },
  "ts": 1710000000000
}
```

字段说明：

- `clientId`：设备指纹（用于唯一标识设备）
- `name`：设备名称（系统设置中的“设备名称”，若为空则使用主机名）
- `service.httpBase`：设备对外可访问的 HTTP 基地址（受“IP 获取方式/手动 IP”影响）
- `ipList`：本机可用 IPv4 列表（私网段）

### 2.7 服务端/管理端实现建议

管理端通常需要两步：

1) 往组播地址发 Probe（端口 10086）  
2) 在 `replyPort` 上监听 UDP，接收设备回包的 Match

管理端若不使用 `replyPort`，也可以直接在发包 socket 的端口上收包（取决于实现方式）。

### 2.8 与 ONVIF 是否冲突

一般不会“协议层面冲突”，因为：

- ONVIF 发现是 WS-Discovery（XML/SOAP），本协议是 JSON；
- 设备只对 `{"t":"probe"}` 这种 JSON 消息响应，收到 ONVIF XML 会忽略。

但如果复用同一组播端口可能带来“端口占用/排障混淆”。因此默认端口已改为 `10086`。

## 3. 注册上报与心跳（客户端模式）

### 3.1 触发条件

仅当 `system.clientMode = true` 时启用。

### 3.2 注册（register）

- `POST {registryBaseUrl}/api/registry/register`
- payload 中包含 `name`（设备名称）、`clientId`（设备指纹）等信息

### 3.3 心跳（heartbeat）

- `POST {registryBaseUrl}/api/registry/heartbeat`
- 默认间隔：**300 秒**（可用 `REGISTRY_HEARTBEAT_INTERVAL_SEC` 覆盖）
- payload 中包含 `name`（设备名称）

## 4. 配置持久化

- 配置文件：`.device-info.json`（系统设置、串口、接收设置、客户端模式等）
- 车牌数据：`data/plates.sqlite3`（SQLite）
